# South Africa Readiness

## Ready Now

- Rand pricing in app copy.
- R99/month pilot offer.
- `+27` phone guidance on signup and WhatsApp orders.
- POPIA-style privacy page.
- Terms page.
- Signup consent checkbox.
- Trial/payment status on shop accounts.
- Manual paid-status command.
- WhatsApp-first outreach copy.

## Language Style

Use simple South African English first.

Good words:

- shop book
- stock
- daily sales
- running low
- expiring
- supplier prices
- reorder
- airtime
- EFT
- WhatsApp orders

Avoid:

- SaaS
- ERP
- AI-powered
- digital transformation
- omnichannel
- merchant intelligence

## First Localized Copy

Hero:

```text
KasiStock: the simple shop book for spaza owners.
```

Subhead:

```text
Know what sold, what is running low, what is expiring, and what to reorder from one phone.
```

Price:

```text
30 days free. Then R99/month. Cancel anytime.
```

Setup:

```text
Need help? Assisted setup is R299 once-off.
```

## Language Rollout

Do not translate the whole app yet.

First add human-reviewed phrases for the first target area:

| Area | First Languages |
|---|---|
| Soweto/Johannesburg | English, isiZulu, Sesotho |
| Cape Town/Khayelitsha | English, isiXhosa, Afrikaans |
| Durban/Umlazi | English, isiZulu |
| Pretoria/Mamelodi | English, Sepedi, Setswana |

## Trust Rules

- Do not claim to be South African if operating from the US.
- Do say the product is built for South African spaza shops.
- Price in rand.
- Support through WhatsApp first.
- Keep cancellation simple.
- Avoid long contracts during pilot.
- Use real screenshots, not generic stock images.

## Outreach Channels

Start with:

- WhatsApp referrals
- Facebook township/small-business groups
- Google Maps/direct messages
- Local setup agents
- Small-business coaches/pages

Do not start with:

- Paid ads
- App stores
- Big influencers
- Broad "Africa" messaging

## Compliance Notes

Pilot minimum:

- Privacy page.
- Terms page.
- Consent checkbox.
- Data deletion/export request path.
- Do not collect unnecessary customer information.
- Do not store sensitive personal data in order notes.

Later:

- Formal POPIA policy review.
- Data processing agreement.
- Hosted database with backups.
- Account self-delete/export.
